商学院排考系统，团队开发完成。
项目架构：spring+springMVC+mybatis+mysql+c3p0数据源。
项目功能：管理员可通过该系统使用上传Excel表格的形式将教室信息和教师信息等录入，然后系统进行排考。
          教师可查看自己监考的考试。
           学生可查看自己参加的考试。
算法处理的问题：1.同一课程安排在同一时间，同一教学楼，并尽量安排在相邻位置。
               2.考试科目的授课老师不可监考该考试，并且在同一教学楼有其他监考，方便送卷。
               3.排除公共课等学校已经排好占用的教室，并在其期间不得占用该教室
               4.尽量将教师的监考时间错开，防止一整天都在监考的情况。
               5.平均各老师的监考次数，并对已经监考四六级的老师减少其监考场次
               6.若需要将一个课程班分到两个教室考试，需使得空余教室位置较少。并且最好将一个专业班的同学归入同一教室
本人负责算法设计，Excel文件上传下载处理及部分后台代码编写
